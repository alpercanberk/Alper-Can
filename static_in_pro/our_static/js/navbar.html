    Chris Kavanagh1 yıl önce
    One thing I noticed in this (https://www.youtube.com/watch?v=ahM4GBZ-6qg) on django templates, is she (presenter) recommends not using {% endblock %} but using the full name of the tag after "endblock". Like so, {% block jumbotron %} -  ............. {% endblock jumbotron %}. Is this a good idea in your opinion? It does seem like it would keep your templates a little more readable, because you might not know which endblock you're referring to if there's more than a few.﻿
    12
    CodingEntrepreneurs1 yıl önce
    +Chris Kavanagh  With code, I believe about being as concise as possible. So, {% block jumbotron %}{% endblock %} is a great solution most of the time. However, if you have a lot of blocks (like 6+) in a template, using {% block jumbotron %}{% endblock jumbotron %} will make it more readable for you and your fellow developers. Each method is acceptable (as noted in Django documentation) but to be more readable, you'd use the {% endblock blockname %} method. That method, is optional for a reason: it's not always needed. Great question.﻿
